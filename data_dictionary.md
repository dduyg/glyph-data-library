# 🔠 Data Dictionary

This document defines every field generated by the *glyph feature pipeline*, describing:
- what it represents  
- how it's computed  
- its data type  
- and possible values

<br>

### 📁 Top-Level Structure
Each processed glyph entry includes:
```json
{
  "id": "",
  "filename": "",
  "glyph_url": "",
  "color": { ... },
  "metrics": { ... },
  "color_harmony": "",
  "mood": "",
  "created_at": { ... }
}
```

<br>

## 🔹 Field Definitions

### id
- **Type:** string  
- **Example:** `"a3f92cd1"`  
- **Description:** A unique 8‑character hash assigned to the glyph using UUID4

---

### filename
- **Type:** string
- **Format:** `{hex_color}_{id}.png`
- **Example:** `"e3b45c_af39dc12.png"`
- **Description:** The output filename created based on dominant color + unique ID for easy sorting

---

### glyph_url
- **Type:** string
- **Description:** Direct CDN URL pointing to the stored glyph image via jsDelivr

---

## 🎨 Color Detection

### color.hex
- **Type:** string (hex color without `#` prefix)
- **Example:** `"a1c2f3"`  
- **Description:** Dominant color of the glyph in hexadecimal representation, extracted using K-means clustering with saturation-weighted sampling

---

### color.group
- **Type:** string (categorical)
- **Possible values:**  
`red` · `orange` · `yellow` · `green` · `blue` · `purple` · `pink` · `brown` · `gold` · `silver` · `gray` · `black` · `white`  
- **Description:** Semantic color group assigned based on HSV values, brightness, and saturation thresholds.
- **Classification logic:**
  - `black`: brightness < 40
  - `white`: brightness > 230 AND saturation < 0.20
  - `gray`: saturation < 0.12 AND 40 ≤ brightness ≤ 230
  - `gold`: 35° < hue < 65°, 120 < brightness < 220, 0.20 < saturation < 0.55
  - `silver`: brightness > 180 AND saturation < 0.18
  - `brown`: brightness < 140 AND saturation > 0.25 AND 15° < hue < 65°
  - Hue-based groups (red, orange, yellow, green, blue, purple, pink) for other cases

---

### color.rgb
- **Type:** array of 3 integers  
- **Range:** `[0-255, 0-255, 0-255]`
- **Example:** `[120, 180, 240]`  
- **Description:** Dominant color extracted using K-means clustering (k=5) with saturation-based weighting to prioritize vivid colors.

---

### color.lab
- **Type:** array of 3 floats `[L, a, b]`
- **Range:** 
  - `L`: 0–100 (lightness)
  - `a`: -128 to +127 (green to red)
  - `b`: -128 to +127 (blue to yellow)
- **Example:** `[58.23, 52.14, 48.76]`
- **Description:** Perceptually uniform CIELAB color space representation for accurate color distance calculations and color theory applications.

---

### color_harmony
- **Type:** string (categorical) 
- **Possible values:** `analogous`, `complementary`, `none`
- **Interpretation:**
  - `analogous`: Dominant and secondary colors are within 30° on the color wheel (harmonious, related colors)
  - `complementary`: Colors are ~180° apart on the color wheel (high contrast, opposing colors)
  - `none`: No clear harmonic relationship (color separation 30°–150°)
- **Description:** Relationship between dominant and secondary colors based on hue angle separation.

---

## 📐 Metrics

### metrics.edge_density
- **Type:** float
- **Range:** 0.0–1.0
- **Example:** `0.1823`
- **Interpretation:** 
  - High values (>0.08) → many edges, complex shapes, detailed linework
  - Low values (<0.02) → smooth, minimal shapes, simple geometry
- **Description:** Proportion of edge pixels detected using Canny edge detection (thresholds: 80, 160) within the glyph mask.
 
---

### metrics.entropy
- **Type:** float
- **Range:** ~0.0–8.0 (theoretical max for 8-bit grayscale)
- **Example:** `5.2341`
- **Interpretation:** 
  - High entropy (>5.0) → textured, detailed, noisy, high information density
  - Low entropy (<2.5) → simple, uniform, minimal variation
- **Description:** Measures informational complexity (*how "busy" the image is*) with Shannon entropy grayscale pixel distribution

---

### metrics.texture
- **Type:** float
- **Description:** LBP (Local Binary Pattern) texture complexity measure.

---

### metrics.contrast
- **Type:** float
- **Description:** Michelson contrast on masked luminance values

---

### metrics.circularity
- **Type:** float
- **Interpretation:** 
  - 1.0 = perfect circle
  - Low values (<0.5) = irregular, elongated, or complex shapes
- **Description:** Shape roundness metric computed from the largest contour, measuring how close the glyph is to a perfect circle.

---

### metrics.aspect_ratio
- **Type:** float
- **Description:** Width-to-height ratio of the largest contour's bounding box.

---

### metrics.edge_angle
- **Type:** float 
- **Description:** Dominant orientation of edges (0–180°).

---

## 🌀 Mood Classification

### mood
- **Type:** string (categorical) 
- **Possible values:**  
`serene` · `calm` · `playful` · `energetic` · `futuristic` · `mysterious` · `dramatic` · `chaotic`
- **Description:** The pipeline classifies glyphs into 8 mood categories using multi-factor analysis:
  - **Serene** - Low entropy, smooth edges, cool tones
  - **Calm** - Moderate entropy, analogous harmony, desaturated
  - **Playful** - Medium-high entropy, varied shapes, bright colors
  - **Energetic** - High saturation, warm tones, dynamic contrast
  - **Futuristic** - Specific aspect ratios, geometric precision
  - **Mysterious** - Low brightness, cool tones, high contrast
  - **Dramatic** - Very high contrast, bold color choices
  - **Chaotic** - Extremely high entropy, complex textures

---

## 🕒 Creation stamp

### created_at.date
- **Type:** string (ISO 8601 date)
- **Format**: `YYYY-MM-DD`
- **Example**: `"2024-11-25"`

### created_at.time
- **Type:** string (ISO 8601 time)
- **Format**: `HH:MM:SS`
- **Example**: `"14:30:22"`
